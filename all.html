当前环境为Mac OS

在 Build with Parameters 时显示当前的pod版本号，并可以在打包时修改。效果如下：

![-w410](media/15960887152430/15960902123521.jpg)

首先安装 Active Choices Plugins，安装完成后打开项目 -> 配置 -> 勾选参数化构建过程 -> 添加参数 -> Active Choices Parameter, 填写 Name，勾选 Groovy Script, 在Script填写脚本

//这里使用项目的绝对路径(absolute path)
def shell = ['/bin/bash', '-c', "grep -E 's.version.*=' /Users/你的用户名(YourName)/.jenkins/workspace/iOS-SDK/Demo/ITOPOD/ITOSDK.podspec"]

def process = shell.execute()
List<String> versions = new ArrayList<String>()
versions.add(process.text + ':selected')

return versions

在 Fallback Script 填写 

return ['error']

构建Shell

这里补充自己写的构建shell脚本，读者可以参考。

export LANG=en_US.UTF-8

#如果填写了PodSpec版本号，修改版本值
if [ $PodSpecVersion ];then
echo "输入版本号:$PodSpecVersion"
sed -i '' -e "s/s.version .*$/s.version          = \'${PodSpecVersion}\'/1" Demo/ITOPOD/ITOSDK.podspec
fi
# 获取podspec版本
VersionNumber=`grep -E 's.version.*=' Demo/ITOPOD/ITOSDK.podspec | tr -d "'a-z= " | sed "s/\.//1"`

echo ${VersionNumber}

if [ ! $VersionNumber ];then
echo "获取版本号错误"
exit -1
fi

# === 打包 === #

# 进入目录
cd ITOSDK_Proj

# 执行打包脚本
xcodebuild -workspace 'ITOSDK_Proj.xcworkspace' -scheme Pack_Full
# 返回上级目录

# 删除并替换文件
rm -rf Demo/ITOPOD/ITOSDK/*

【转载】树莓派配置v2ray客户端

2020/07/02 17:57 下午

点击打开原文链接，本文有修改。

下载安装

使用一键安装脚本安装，参考v2ray官方教程-下载安装

bash <(curl -L -s https://install.direct/go.sh)

可能会出现网络错误Failed to fetch release information. Please check your network or try again.

这就需要从github下载安装包, 先从v2ray-core/releases将安装包v2ray-linux-arm.zip下载到树莓派, 使用--local参数从本地安装

wget https://github.com/v2ray/v2ray-core/releases/download/v4.25.1/v2ray-linux-arm.zip
wget -O v2ray_install.sh  https://install.direct/go.sh
chmod +x v2ray_install.sh
sudo ./v2ray_install.sh --local v2ray-linux-arm.zip

配置v2ray客户端

建议:如果已经有V2rayU一类的客户端，且已有代理服务器，可以打开V2rayU客户端菜单服务器设置选择你已有的服务器，选择导入模式，复制配置好的json。在cat > /etc/v2ray/config.json << EOF 命令后直接粘贴。

sudo su - root
cat > /etc/v2ray/config.json << EOF
{
  "inbounds": [{
    "port": 1080,
    "listen": "127.0.0.1",
    "protocol": "socks",
    "settings": {
      "udp": true
    }
  }],
  "outbounds": [{
    "protocol": "vmess",
    "settings": {
      "vnext": [{
        "address": "server ip",
        "port": 10086,
        "users": [{ "id": "b831381d-6324-4d53-ad4f-8cda48b30811" }]
      }]
    }
  },{
    "protocol": "freedom",
    "tag": "direct",
    "settings": {}
  }],
  "routing": {
    "domainStrategy": "IPOnDemand",
    "rules": [{
      "type": "field",
      "ip": ["geoip:private"],
      "outboundTag": "direct"
    }]
  }
}
EOF


# address改为服务端ip， port改为服务端端口, id改为服务端用户ID

开启v2ray服务

sudo systemctl start v2ray

#查看v2ray服务状态, Active: active (running)代表程序运行中

使用ProxyChains-NG配置全局代理

git clone https://github.com/rofl0r/proxychains-ng.git
cd /home/pi/proxychains-ng/
./configure && make && make install

添加配置:

cp ./src/proxychains.conf /etc/proxychains.conf

vim /etc/proxychains.conf

# 将最后的 socks4 127.0.0.1 9095 改为 socks5  127.0.0.1 1080

在需要使用代理的命令前面加上proxychains4, 如下

curl ip.sb
xxx.xxx.xxx.xxx
# 这里应该是你当前的ip

proxychains4 curl ip.sb
x.x.x.x
# 这里是代理的ip

Chrome浏览器使用代理

1. 首先通过命令行启动Chrome浏览器

chromium-browser --proxy-server="https=socks5://127.0.0.1:1080;http=socks5://127.0.0.1:1080"

注：这种方式不可以在SSH中启动，因SSH中无界面导致失败。

命令行-单元测试

使用Cocoapods有workspace 必须加-workspace，注意使用引号及完整的文件名， [-only-testing:<test-identifier>] test-identifier格式说明 TestTarget[/TestClass[/TestMethod]]

xcodebuild test -workspace 'opensdk.xcworkspace' -scheme 'opensdk' -destination 'platform=iOS Simulator,name=iPhone 8,OS=13.5'
xcodebuild test -workspace 'opensdk.xcworkspace' -scheme 'opensdk' -destination 'platform=iOS Simulator,name=iPhone 8,OS=13.5' -only-testing:'opensdkTests'
xcodebuild test -workspace 'opensdk.xcworkspace' -scheme 'opensdk' -destination 'platform=iOS Simulator,name=iPhone 8,OS=13.5' -only-testing:'opensdkTests/LoginTests/testLoginExample'

省略了class
xcodebuild test -workspace 'opensdk.xcworkspace' -scheme 'opensdk' -destination 'platform=iOS Simulator,name=iPhone 8,OS=13.5' -only-testing:'opensdkTests/testLoginExample'

Shell 笔记

2020/06/17 15:09 下午

获取podspec版本号

VersionString=`grep -E 's.version.*=' xxx.podspec`
VersionString=${VersionString##*=}
VersionNumber=`tr -d "'" <<<"$VersionString"`
echo "current version is ${VersionNumber}"

20200730更新
经过几天的Jenkins折腾，借助管道，获取版本号的命令现在仅需要一行.
首先使用grep命令，读取文件内容并过滤到s.version这一行；然后使用tr命令去掉单引号、字母、空格、等号；最后使用sed命令，替换第一个点。

grep -E 's.version.*=' xxx/xxx.podspec | tr -d "'a-z= " | sed "s/\.//1"
