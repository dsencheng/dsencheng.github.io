Jenkins

2019/08/13 10:52 上午

准备工作

  1. 开发环境，如Java、Ruby、MacOS等。
  2. 开发工具，如Xcode、Android Studio、IntelliJ IDEA等。
  3. 证书管理，如推送证书、商店证书、手机配置文件等。
  4. 其他依赖工具，如Git、Fastlane、Cocoapods、Maven等。

准备完成后，创建新项目或使用已有项目，打包一次验证是否可以成功打包。

安装

Jenkins中文安装文档

1、推荐安装方式，安装完成自动运行。不推荐PKG方式安装

brew install jenkins-lts

# 启动/停止/重启

brew services start jenkins-lts
brew services stop jenkins-lts
brew services restart jenkins-lts

2、简单方式，下载jenkins-lts.war，cd到文件所在目录，使用命令启动

java -jar jenkins-lts.war

3、浏览器访问: http://localhost:8080 或者 http://10.0.1.147:8080 按引导教程完成配置。若局域网内无法访问，参考下面的问题记录。

Jenkins构建

  1. 创建一个FreeStyle的任务。
  2. 新任务的配置工作。
  3. 构建。

任务配置介绍

1.General
    设置文件存留天数、数量等。
2.源码管理
    git、svn
3.构建触发器
    提供webhook，轮询，定时等触发构建条件，便于打包自动化。
4.构建环境
    设置构建的参数、打包证书(需要插件)等。
5.构建
    提供shell、插件等多种构建方式，可以执行多个构建方式，并且有顺序。比如先执行shell，然后执行插件构建。
6.构建后操作
    提供通知，邮件，推送等构建完成后任务。

问题记录

无法更新插件

管理Jenkins > 插件管理 > 高级
把URL里的地址https改为http

局域网内，无法通过ip+port访问

关闭Jenkins服务，打开文件夹 /usr/local/Cellar/jenkins/,找到homebrew.mxcl.jenkins.pist这个文件.

--httpListenAddress=127.0.0.1
修改为
--httpListenAddress=0.0.0.0

启动Jenkins，局域网其他用户就可以访问了。

fastlane: command not found

参照安装方法添加路径sudo gem install fastlane -n /usr/local/bin，在Jenkins > 节点管理 > 配置(右侧⚙图标) > 设置环境变量。
查看当前环境变量,

echo $PATH
